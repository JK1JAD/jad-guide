事前準備

  • GitHubにログイン済みであること
  • ユーザー名（例：JK1JAD）を確認

JADの一言：まずは周波数合わせから。落ち着いていけばQSO成功💪

① リポジトリを作成

  1. 右上「＋」→ New repository
  2. Repository name：例 jad-guide
  3. Public を選ぶ（Pages公開のため）
  4. Add a README にチェック（任意）
  5. Create repository をクリック

② index.html を作る

メモ帳などで次の内容を作り、index.html という名前で保存して、リポジトリにアップロードします。

<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>JADガイド</title>
</head>
<body>
  <h1>JADガイド公開テスト</h1>
  <p>こんにちは！これは最初のGitHub Pagesサイトです 🎉</p>
</body>
</html>

GitHub上では、「Add file → Upload files」 からドラッグ＆ドロップでOK。

③ GitHub Pages を有効化

  1. リポジトリの Settings → 左メニュー Pages
  2. Branch：main ／ フォルダ：/(root)
  3. Save をクリック（数分でデプロイ完了）

④ 公開を確認

  • 公開URL（例）：https://JK1JAD.github.io/jad-guide/
  • 表示されない場合は トラブルシューティング へ

画像の入れ方

  1. リポジトリに images フォルダを作成（「Add file → Create new file」で images/.keep として作る方法が簡単）
  2. 画像（PNG/JPG）を images にアップロード
  3. HTMLに記述： 
    <img src="images/screenshot-pages.png" alt="Pages設定画面の例" width="640">

※ 画像ファイル名は半角英数・小文字・スペースなしが吉（電波はクリアに）。

トラブルシューティング

  • 404になる／古い内容が出る：数分待ってリロード。Ctrl+F5 でキャッシュクリア。
  • 真っ白：ファイル名が index.html か確認。拡張子 .txt になっていないか。
  • 日本語が文字化け：<meta charset="UTF-8"> を入れて、UTF-8で保存。
  • デプロイに失敗：リポジトリ右側の「Environments → github-pages」からログを確認。

用語ミニ辞典

  • リポジトリ：プロジェクトの“作業場”。
  • コミット：変更の保存（ログ記録）。
  • プッシュ：ローカルからGitHubへ送信。
  • GitHub Pages：リポジトリをWebサイトとして公開できる機能。
